# Simple x4 — Template 3

Date: 2026-06-08
Status: Approved (decisions in chat)

## Goal

Add a third card-template option to the simple x4 tab, with a deep-navy
title band + bright-red full-width CTA pill + yellow CTA text. Sits
alongside Templates 1 and 2; operators pick via the existing per-row
dropdown (becomes `'' | '1' | '2' | '3'`).

## Mockup source

User-supplied 1080×1080 PNG (chat 2026-06-08): square photo on top
(~75%), a thin red separator line, a deep-navy band (~17%) with the
white bold all-caps headline, and a bright-red full-width pill (~8%) at
the bottom with bold yellow "DISCOVER MORE >>".

## Decisions (chat confirmations)

- **Add T3 alongside T1+T2** — dropdown becomes 1/2/3; no removals.
- **Hardcoded colors** — navy band, red pill, yellow text bake into
  `_DESIGNS[TEMPLATE_3]`. Future tunability is a 1-line constant edit.
- **Default CTA = per-language "Learn More >>"** — reuses the existing
  30-language table in `pipeline/cta_defaults.py`. Operator can still
  type "DISCOVER MORE >>" in the CTA cell to match the mockup verbatim.

## Color palette (hardcoded)

| Element | RGB |
|---|---|
| Title band background | `(15, 30, 55)` — deep navy |
| Title text | `(255, 255, 255)` — white |
| Separator line below image | `(230, 30, 35)` — red |
| CTA pill background | `(230, 30, 35)` — red |
| CTA pill text | `(255, 215, 0)` — yellow |

## Layout (proportions)

- Image cover: top 75% of canvas (cover-cropped to width × image_h)
- Red separator: 2px below image
- Deep navy band: 17% of canvas height, containing the title
- Red full-width pill: 8% of canvas height, anchored to bottom (touching
  the navy band, no gap)

## Security / safety

No new attack surface. Template_id is server-side coerced to the
allow-list `{'', '1', '2', '3'}`; any unknown value falls through to
"no overlay", matching the existing pattern. The renderer is pure
Pillow CPU — no network, no FS writes outside the in-memory PNG buffer.

## Observability

`card_render_start` / `card_render_done` log lines already emit the
template id, so T3 renders are automatically distinguishable in HF
logs without any new instrumentation.

## Settings audit (rule 15)

Two new settings get registered:
- `card_template_3_default_cta` (default empty → per-language fallback)
- `card_preview_url_template_3` (default empty → preview row stays blank
  until the operator runs `upload_template_previews.py`)

Both sit in the same admin settings panel as the T1/T2 equivalents.
Grouped by template number, labels match the existing pattern.
